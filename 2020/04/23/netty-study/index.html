I/O模型

BIO blocking i/o

  • 阻塞并同步，一个连接一个线程，如果无法立即返回，则会导致阻塞。

  • 一个客户端创建一个线程与之通信，如果连接的客户端暂时没有数据发送，会一直阻塞在read，占用服务器资源。

  • 适用：连接数目比较小且固定的架构，服务器要求较高。

    BIO 传统IO

    简单流程

    • 服务端启动一个websocket
    • 客户端启动socket进行通讯，（一般）服务器端需要对每一个客户端建立一个线程与之通讯
    • 客户端发出请求后，先咨询服务器是否有空闲线程响应，如果没有则等待或遭到服务端拒绝
    • 如果有响应，会一直等待到服务端处理完毕

NIO

  • 非阻塞 同步，一个线程处理多个（通道）请求
  • 面向缓冲区 – 数据写入到缓冲区中，需要时再可在缓冲区中前后移动， – 实现非阻塞
  • 适用：连接数目多且连接比较短的架构，比如：聊天服务器、弹幕系统等。
  • 三个组件：
    • selector – 选择通道
    • channel – 通过channel read/write
    • buffer – 程序与buffer进行数据读写
    • 组件模型图：
      • 更正图像
      1. 每个channer对应一个buffer
      1. 每个线程对应一个线程，一个线程可以对应多个selector
      2. 程序切换到哪一个channel由Event决定
      3. 可以有多个channel注册到selector上
      4. selector根据不同evnet切换不同通道
      5. Buffer底层是数组，可以看作内存块
      6. 数据的读写都是通过buffer的，buffer 的 flip() 方法可以转换标志
      7. channel是双向的，可以返回底层操作系统的情况

AIO

  • JDK1.7之后存在，异步非阻塞。
  • 适用：连接数目多且连接比较长的架构，比如：相册服务器等。充分调用OS参与操作，但是应用还不是很广泛。

NIO VS BIO

  1. NIO是块读写，BIO是流读写，块的处理效率比流的效率更高
  2. NIO非阻塞，BIO阻塞
    • （NS） buffer 当有足够的数据读写时，线程才会处理，其余时间线程可以处理其他内容，所以是‘异步’的，而BIO每一个客户端连接，服务端就需要开启一个线程处理。
  4. NIO 一个线程管理多个客户端，当一个channel有事件发生时，selector选中这个通道。

NIO三组件

Buffer（缓冲区）

缓冲区本质上是可读写的内存块，底层是数组，Channel提供从文件、网络读取数据的渠道，但是读取和写入都必须经过Buffer

2.buffer四个参数

  1. capacity - 容量，不可变，创建buffer时指定
  2. limit - 可更改，可读取的索引最大值，不可以对超过极限的位置进行操作 （如limit设置为3，则只能操作索引为3之前的数据）
  3. position <= capacity 读取的位置，可指定，每次操作数据都会改变
  4. mark - 标志位
